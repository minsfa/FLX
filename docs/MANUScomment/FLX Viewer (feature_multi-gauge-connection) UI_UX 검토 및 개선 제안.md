# FLX Viewer (feature/multi-gauge-connection) UI/UX 검토 및 개선 제안

**작성자**: Manus AI
**날짜**: 2026년 2월 17일

## 1. 개요

본 문서는 `feature/multi-gauge-connection` 브랜치에 추가된 WPF 기반의 **FLX Viewer** 애플리케이션에 대한 심층적인 UI/UX 분석 결과를 담고 있습니다. 현재 구현된 다크 테마 기반의 UI는 기능적으로 훌륭하지만, 전문성과 사용자 경험을 한 단계 끌어올리기 위한 몇 가지 개선 영역이 존재합니다. 본 제안은 색상, 디자인 패턴, 일관성, 접근성 측면에서 구체적이고 실행 가능한 개선 방안을 제시합니다.

## 2. 종합 분석

### 2.1. 잘된 점 (Strengths)

- **명확한 3단 레이아웃**: [디바이스] - [스터디] - [차트]로 이어지는 구조는 정보의 흐름과 일치하여 직관적입니다.
- **다크 테마 적용**: 장시간 모니터링 환경에 적합한 다크 테마를 기본으로 채택했습니다.
- **상태의 다중 표현**: 연결 및 스터디 상태를 색상, 아이콘, 텍스트로 동시에 표현하여 명확성을 높였습니다.
- **핵심 기능의 시각화**: ScottPlot을 활용한 실시간 차트 기능은 데이터 분석의 핵심적인 역할을 훌륭히 수행합니다.

### 2.2. 주요 개선 필요점

1.  **디자인 일관성 부족**: 메인 윈도우와 Flux 계산 창 간의 테마(다크/라이트)가 통일되지 않았으며, 버튼, 텍스트 상자 등 기본 컨트롤들이 일관된 스타일을 갖추지 못했습니다.
2.  **색상 대비율 문제**: 일부 텍스트와 배경색 간의 명도 대비가 WCAG 접근성 표준에 미치지 못하여 가독성이 저하됩니다. (예: 헤더 텍스트 `#14427B`)
3.  **인터랙션 피드백 부재**: 버튼의 눌림(Pressed) 효과나 입력 필드의 포커스(Focus) 효과 등 사용자의 행동에 대한 시각적 피드백이 부족하여 UI가 정적으로 느껴집니다.
4.  **아이콘 시스템의 부재**: 이모지(Emoji)와 유니코드 문자를 아이콘으로 혼용하고 있어, 플랫폼 간 렌더링 차이나 디자인 확장성에 한계가 있습니다.

## 3. 색상 스키마 개선 제안

현재 색상 팔레트는 일관성이 부족하고 일부 색상은 대비율이 낮습니다. 의미론적이고 체계적인 색상 시스템을 도입할 것을 제안합니다.

### 3.1. 제안 팔레트 (App.xaml)

의미와 역할에 따라 색상을 재정의하고, 모든 색상을 `App.xaml`에서 중앙 관리하여 일관성을 확보합니다.

| 역할 | 색상명 | 16진수 코드 | 용도 |
|---|---|---|---|
| **기본/정보** | `InfoColor` | `#2196F3` | 기본 버튼, 정보성 텍스트 |
| **성공/준비** | `SuccessColor` | `#4CAF50` | 연결 성공, 준비 상태, 시작 버튼 |
| **위험/오류** | `DangerColor` | `#F44336` | 연결 끊김, 오류, 중지/삭제 버튼 |
| **경고** | `WarningColor` | `#FFC107` | 재연결 시도, 주의 필요 상태 |
| **녹화 중** | `RecordingColor` | `#F44336` | 녹화 중임을 알리는 강조 색상 (빨간색) |
| **배경** | `BackgroundColor` | `#1E1E1E` | 앱 전체 배경 |
| **표면** | `SurfaceColor` | `#2D2D2D` | 카드, 패널 등 구분되는 영역 배경 |
| **주요 텍스트** | `TextColor` | `#FFFFFF` | 본문 텍스트 |
| **보조 텍스트** | `TextSecondaryColor` | `#B0B0B0` | 보조 정보, 설명 텍스트 |

### 3.2. 구현 예시

```xml
<!-- App.xaml -->
<Color x:Key="SuccessColor">#4CAF50</Color>
<SolidColorBrush x:Key="SuccessBrush" Color="{StaticResource SuccessColor}"/>

<!-- MainWindow.xaml -->
<Button Content="Start" Background="{StaticResource SuccessBrush}" ... />
```

## 4. UI 디자인 시스템 구축 제안

일관된 사용자 경험을 위해 공통 스타일과 템플릿을 정의하는 디자인 시스템을 구축해야 합니다.

### 4.1. 컨트롤 스타일 통일

모든 기본 컨트롤(Button, TextBox, CheckBox, Slider 등)에 다크 테마에 맞는 공통 스타일을 적용합니다. 특히 다음과 같은 효과를 추가하여 인터랙션을 강화해야 합니다.

- **호버(Hover) 효과**: 마우스를 올렸을 때 색상 변경
- **눌림(Pressed) 효과**: 클릭 시 크기 변화나 색상 변경
- **포커스(Focus) 효과**: 키보드로 선택 시 테두리 강조
- **비활성화(Disabled) 상태**: 명확한 비활성화 스타일

**권장 사항**: 모든 스타일을 별도의 `Styles.xaml` 파일로 분리하고 `App.xaml`에 병합하여 관리합니다.

### 4.2. 아이콘 라이브러리 도입

이모지 대신 벡터 기반의 아이콘 라이브러리를 사용하여 디자인의 일관성과 확장성을 확보합니다.

- **추천 라이브러리**: **MahApps.Metro.IconPacks** (다양한 아이콘 팩 포함, 사용 용이)
- **설치**: `Install-Package MahApps.Metro.IconPacks`

#### 구현 예시

```xml
<!-- xmlns:iconPacks="http://metro.mahapps.com/winfx/xaml/iconpacks" -->

<!-- 기존 -->
<Button Content="🗑" ... />

<!-- 개선 -->
<Button>
    <iconPacks:PackIconMaterial Kind="Delete" Foreground="{StaticResource DangerBrush}"/>
</Button>
```

### 4.3. 타이포그래피 체계화

정보의 위계를 명확히 하기 위해 폰트 크기와 굵기를 체계적으로 정의합니다.

| 스타일명 | 크기 | 굵기 | 용도 |
|---|---|---|---|
| **H1** | 24pt | Bold | 창 제목 (예: Flux Calculation) |
| **H2** | 18pt | SemiBold | 현재 압력, 주요 결과 값 |
| **Body** | 14pt | Normal | 일반 텍스트, 버튼 |
| **Caption** | 12pt | Normal | 보조 정보 (포트명, ID) |

## 5. 주요 화면별 개선 제안

### 5.1. MainWindow.xaml

- **디바이스 카드**: 호버 시 배경색을 미세하게 밝게 변경하고, 그림자 효과를 추가하여 입체감을 줍니다.
- **상태 표시**: 'Recording' 상태일 때, 상태 배지뿐만 아니라 카드 전체에 얇은 빨간색 테두리를 추가하여 주목도를 높입니다.
- **버튼**: 모든 버튼에 통일된 스타일과 아이콘을 적용합니다.

### 5.2. FluxCalculationWindow.xaml

- **테마 통일**: 창의 배경과 컨트롤들을 메인 윈도우와 동일한 다크 테마로 변경합니다.
- **입력 필드**: `TextBox`, `Slider` 등에 다크 테마용 스타일을 적용하여 흰색 배경 문제를 해결합니다.
- **결과 섹션**: 계산된 결과(Flux, Permeance)를 더 눈에 띄는 색상과 큰 폰트로 강조합니다.

## 6. 결론 및 실행 계획

FLX Viewer는 이미 강력한 기능을 갖추고 있으며, 제안된 UI/UX 개선 사항들을 적용한다면 사용 편의성과 전문성을 크게 향상시킬 수 있습니다. 아래와 같은 단계적 실행 계획을 추천합니다.

- **1단계 (즉시)**: `App.xaml`에 제안된 색상 시스템을 정의하고, 명도 대비가 낮은 색상들을 우선적으로 교체합니다.
- **2단계 (단기)**: 모든 컨트롤에 대한 공통 스타일을 `Styles.xaml`에 정의하고, 아이콘 라이브러리를 도입하여 이모지를 대체합니다.
- **3단계 (중기)**: `FluxCalculationWindow`의 테마를 통일하고, 앱 전체의 타이포그래피 체계를 적용합니다.

첨부된 `color_analysis.md`와 `ui_design_analysis.md` 파일은 각 분석 영역에 대한 더 상세한 내용과 코드 예시를 포함하고 있으니, 실제 개발 과정에서 참고하시기 바랍니다.
